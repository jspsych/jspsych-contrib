# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

To report a security concern, use GitHub's private vulnerability reporting feature if it is available for this repository. If private vulnerability reporting is not available, contact a repository maintainer directly and avoid sharing vulnerability details publicly until maintainers have had time to investigate.

## Response Timeline

We aim to acknowledge security reports within 72 hours.

We aim to investigate and resolve confirmed vulnerabilities within 90 days, depending on severity, complexity, and maintainer availability.

## Disclosure Process

Maintainers will review reported vulnerabilities, assess severity, and coordinate any necessary fixes before public disclosure. Please avoid publicly disclosing a vulnerability until maintainers have had a reasonable opportunity to investigate and release a fix.

## Supported Versions

Security updates are generally provided for actively maintained versions of packages in this repository. Users should update to the latest available release when security fixes are published.
